ESIF data cleaning

Co

Sada skriptů na transformaci geografických údajů o projektech EU fondů (ESIF) v ČR z obodbí 2014-2020. Cíl je rozpadnout každý projekt na obecní úroveň podle informací patrných buď ze zadaných geografických údajů, nebo z textových polí a informací o projektu a příjemci.

Jak s tím pracovat

  • dev.Rmd a online: základní informace k architektuře (anglicky)
  • how-to-repeat.Rmd a online: kompletní návod na zopakování postupu s novými daty

Co je kde

Konfigurace

  • celková konfigurace: config.yml
  • konfigurace webového výstupu: _site.yml
  • konfigurace Netlify: .netlify/

Sofistikovanější vyyužití config.yml viz dokumentaci a soubor .Rprofile.

Build skripty, viz návod v how-to-repeat.Rmd online

  • build-and-render.R: celkový build v R
  • build-and-render.sh: totéž v Bashi

R a Rmarkdown skripty

  • 0N_*.R: skripty, které načítají vstupní data a externí metadata
  • NN_*.Rmd: validace, transformace a export dat; též generuje webovou dokumentaci
  • ostatní *.Rmd další dokumentace a nástroje, které se též promítnou ve webové dokumentaci

Výstup

  • Arrow dataset v data-output
  • Excel (chunked) v data-export
  • dokumentace exportu v codebook.Rmd a online
  • web na https://esif-data-cleaning.netlify.app

How to reproduce

First, install all the necessary R packages: run renv::restore() (this only needs to be done once and only rerun if the renv.lock file changes.)

Then:

  • load and transform data and metadata by running numbered R scripts in order. This relies on input XLSX files being in data-input, in a subdirectory set in the pth variable in 01_load-data.Rmd
  • run rmarkdown::render_site() or run numbered RMarkdown scripts in order

or run build-and-render.R to run both these steps.

To subsequently build the site afresh and deploy to Netlify

  • on a Linux/Mac machine, run build-site-and-deploy.sh; this requires the Netlify CLI
  • or deploy the docs folder to Netlify using other means.

Environment reproducibility is maintained using renv. This means that upon running R in the project folder, all the necessary packages will be installed in a local library by running renv::restore() (in fact, installed in a cache from which they can be reused, and linked into the local project library where they take up no space.) Should this fail, you can run renv::deactivate() and install the packages manually, though you might end up with different versions.
